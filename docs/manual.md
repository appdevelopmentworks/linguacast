# linguacast 導入マニュアル

海外のポッドキャスト・講義動画を、ローカルAIで日本語の音声に変換するデスクトップアプリです。
このマニュアルでは、必要なソフトの導入からアプリの使い方までを説明します。

---

## 1. 動作環境

| 項目 | 推奨 |
|---|---|
| OS | Windows 10 / 11（64bit）、macOS（Apple Silicon） |
| GPU | NVIDIA GPU（VRAM 8GB以上を推奨。文字起こしと翻訳が高速になります） |
| メモリ | 16GB 以上 |
| ディスク | 空き 50GB 以上（AIモデルの保存に使用） |
| ネットワーク | 動画の取得と初回のモデルダウンロード時に必要 |

GPU が無くても動作しますが、文字起こしが遅くなります。非力なマシンでは
クラウドAPI（OpenRouter / Google Cloud TTS）へのフォールバックを利用できます（後述）。

---

## 2. 必要なソフトのインストール

linguacast は以下の外部ソフトを利用します。**(1)〜(2) は必須**、(3)〜(4) はローカルAI処理に使います（無い場合は警告が出ますがアプリは起動します）。

### (1) ffmpeg（必須・音声処理）

PowerShell で:

```powershell
winget install Gyan.FFmpeg
```

インストール後、PowerShell を開き直して `ffmpeg -version` が表示されればOKです。

### (2) yt-dlp（必須・動画取得）

```powershell
winget install yt-dlp.yt-dlp
```

`yt-dlp --version` が表示されればOKです。

### (3) Ollama（ローカルLLM — 翻訳・要約に使用）

1. https://ollama.com/download からインストーラーを取得して実行します。
2. インストール後、PowerShell で翻訳用・要約用モデルを取得します:

```powershell
ollama pull translategemma:27b   # 翻訳用（既定モデル・約17GB）
ollama pull qwen3.6              # 要約・台本生成用（約23GB。日本語が自然）
```

> 💡 VRAM が少ないマシンでは、より小さいモデル（例: `gemma4`）でも動作します。
> アプリの設定画面でモデルを切り替えられます。

Ollama は常駐サービスとして動作します（タスクトレイにアイコンが表示されます）。
モデルをメモリに保持して高速化するため、環境変数の設定を推奨します:

```powershell
[Environment]::SetEnvironmentVariable("OLLAMA_KEEP_ALIVE", "30m", "User")
```

### (4) 音声合成について — 実は追加インストール不要で始められます

日本語の読み上げは、**何も設定しなくても無料の Edge TTS（Microsoft のニューラル音声・
APIキー不要・インターネット接続のみ必要）で動作します**。まずはこのまま使い始めてOKです。

**ずんだもん等のキャラクター音声を使いたい場合**は VOICEVOX を導入します（任意）:

1. https://voicevox.hiroshiba.jp/ からダウンロードしてインストール。
2. linguacast を使う前に VOICEVOX を起動（内部エンジンが `:50021` で待ち受けます）。
3. アプリのヘッダーの「● VOICEVOX」ランプが緑になれば認識されています。
   キャラクターはドロップダウンで選択できます。

読み上げエンジンは **VOICEVOX → Edge TTS → Google Cloud TTS** の順で自動選択され、
どれが使われたかは結果カードに表示されます。
※ 吹き替えモード（動画への同期）のみ VOICEVOX が必要です。

### (5) 任意: クラウドフォールバック用のAPIキー

ローカルLLMが使えない環境向けです。アプリの「⚙ 設定」から登録します（OSのキーチェーンに安全に保存され、ソースや設定ファイルには残りません）。

- **OpenRouter**（翻訳・要約のクラウド実行）: https://openrouter.ai/ でキーを発行し、
  設定画面でキーを登録。モデルは**ドロップダウンから選択**できます（入力で絞り込み可）。
- **Google Cloud TTS**（高品質クラウド音声・任意）: Google Cloud で Text-to-Speech API を
  有効化し、APIキーを設定画面に登録。Edge TTS で十分な場合は不要です。

## 3. linguacast のインストール

配布されたインストーラー（`linguacast_0.1.0_x64-setup.exe`）を実行し、指示に従ってください。

- **アプリ本体**: 標準のアプリフォルダにインストールされます（スタートメニューに登録）。
- **生成物（音声・字幕・動画）の保存先**: `ダウンロード\linguacast\` フォルダです。
  各処理（ジョブ）ごとにサブフォルダが作られ、アプリの「📁 開く」ボタンからいつでも開けます。

> 開発者向け: インストーラーは `npm run tauri build` で作成します。
> 出力先は `src-tauri\target\release\bundle\nsis\`（.exe）と `bundle\msi\`（.msi）です。

## 4. 使い方

### 基本の流れ（一括実行）

1. VOICEVOX と Ollama が起動していることを確認（画面上部に警告が無ければOK）。
2. 画面上部のボックスに YouTube 等の URL を貼り付ける
   — または、手持ちの音声・動画ファイルを**ドロップゾーンにドラッグ＆ドロップ**。
3. 出力モードを選ぶ:
   - **全訳 × 字幕のみ** → 和訳SRT（字幕ファイル）を生成
   - **全訳 × 吹き替え** → 元動画に日本語音声を同期・多重化した動画を生成
   - **要約** → 内容を要約した日本語ポッドキャスト音声を生成
4. **「▶ 一括実行」** をクリック。進捗バー（%表示）で状況を確認できます。
5. 完成したら:
   - **📁 開く** — 生成物のフォルダをエクスプローラーで開く
   - **📱 QRで送る** — 同一Wi-Fi上のスマホでQRを読み取り、ブラウザで再生・ダウンロード

### プリセットチャンネル

URL欄の下に、良質な英語チャンネル（AI・プログラミング・投資）のプリセットが表示されます。
クリックすると直近の動画一覧が開き、選ぶだけでURLが入力されます。
プリセットは「⚙ 設定」で追加・削除・並べ替えができます。

### 手動ステップ実行 / 途中再開

- 「取得のみ（手動ステップ）」から始めると、文字起こし→和訳→…を1段ずつ実行できます。
- 処理は段階ごとに保存されるため、途中で失敗・中断しても
  **「最近のジョブ」から選んで続きから再開**できます（長時間動画でも最初からやり直しません）。

### QR配信の初回設定

初めて「QRで送る」を使うと、Windows ファイアウォールの確認が表示されることがあります。
**「プライベートネットワーク」にチェックして「アクセスを許可」**してください。
ダウンロードURLは推測不能なトークン付きで、60分で失効します。

## 5. よくあるトラブル

| 症状 | 対処 |
|---|---|
| 「VOICEVOX が起動していません」 | そのままでも Edge TTS（無料）で読み上げ可能。キャラクター音声を使うには VOICEVOX を起動 |
| 「利用可能な翻訳バックエンドがありません」 | Ollama が起動しているか確認（`ollama list` でモデル一覧が出るか）。またはOpenRouterキーを設定 |
| 翻訳モデルが見つからない | `ollama pull translategemma:27b` を実行。アプリは利用可能なモデルに自動で切り替えて動作を継続します（使用モデルは画面に表示） |
| 文字起こしが遅い | GPU が使われているか確認（NVIDIAドライバ最新化）。初回はモデル（約3GB）のダウンロードで時間がかかります |
| QRを読んでも開けない | ① PCとスマホが同一ネットワークか確認（PCが有線・スマホがWi-Fiでも同じルーターならOK）② **PCのネットワークプロファイルが「パブリック」になっていると全ブロックされます**。設定 → ネットワークとインターネット → イーサネット/Wi-Fi → 「プライベート」に変更 ③ ファイアウォールで linguacast を許可 |
| yt-dlp のエラー | `winget upgrade yt-dlp.yt-dlp` で最新化（YouTube側の変更への追従が必要な場合があります） |
